#### 算数操作符：
  
`+` `-` `*` `/` `%`

包含加、减、乘、除、取模等运算

- 其中除法分为`整形除法`与`浮点型除法`

    >eg 1.1:
    1 / 2 = 0
    // 这里是整形除法，无论结果有无小数部分都只返回所得值的整数部分
    // 注意返回的是整数部分而不是进行四舍五入

    >eg 1.2:
    1.0 / 2 = 0.5
    // 只要有一个数字为浮点型，那么进行除法运算时就为浮点型运算，所得值为浮点型

    - 取模操作符 ` % ` 两端必须为整数

#### 移位操作符
` << `  -- 左移操作符
` >> `  -- 右移操作符

- 移位操作符移动的是二进制位

    >eg 2.1 : 
    int a = 7;
    int b = a << 1;

    这里为`左移操作符`，移动的是二进制位

    - 什么是二进制位？

        数值可以用二进制、八进制、十进制、十六进制表示

        十进制的每一位都是使用0 ~ 9组成

        而二进制位只使用 0 ~ 1组成

        但是整数的二进制表示有三种

        - HEX，英文全称 Hexadecimal，表示十六进制

        - DEC，英文全称 Decimal，表示十进制

        - OCT，英文全称 Octal，表示八进制

        - BIN，英文全称 Binary，表示二进制

        为`原码`、`反码`、`补码`

        - ==正整数的三码合一==

        - 负数三码需要计算

        > eg 2.2:
        D7 == B000000000000000000000000000000000000111
        这是十进制7的`原码`
        其中第一位为符号位，整形7有4个字节共计32位
        它的三码相同

        >eg 2.3:
        D-7 == B100000000000000000000000000000000000111
        这里是-7的`原码`
        保持原码的符号位不变，其他位==按位取反==，得`反码`
        `反码`B1111111111111111111111111111111111111000
        将反码加一得`补码`
        `补码`B1111111111111111111111111111111111111001

        - 整数在内存中存储的是`补码`

        - 移位时移动的是内存中存储的值，即`补码的二进制序列`

##### 左移操作符

左移相当于将B1`1111111111111111111111111111111111110010`

即==向左丢去，向右补零==

所以例题中b将会赋值为`14`，` a `值不变(7 + 8 - 1)

- 左移对于任何数的效果为`x 2`

> eg
>  7  << =  14
>  -7 << = -14

- 浮点数的左移？
    左移右移只支持整数，不支持浮点数！！！

##### 右移操作符

1. 算数移位
   
   >eg 2.3:
   int a = 7;
   int b = a >> 1

   补码右边丢弃，左边补`符号位`

2. 逻辑移位
   
   >eg 2.3:
   int a = 7;
   int b = a >> 1

   补码右边丢弃，左边补` 0 `

- vs编译器采用`算数右移`

- 右移即==向右丢弃，向左补0/数==

- 算数右移的效果为' / 2'，取整方式为向下取整

    如 -7 >> 1 = -4






            








==移位操作符操作数只能是整数==
